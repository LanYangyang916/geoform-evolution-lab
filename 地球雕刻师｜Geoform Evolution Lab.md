请帮我从零搭建一个可运行的前端 MVP 网站，项目名称为：

# 地球雕刻师｜Geoform Evolution Lab
副标题：互动式河流地貌 3D 图鉴馆

这是一个面向中学地理学习的 Three.js 互动教学网站。第一版只聚焦“河流地貌”，包含四个可交互的 GLB 模型：

- V 型谷：`/public/models/river/v-valley.glb`
- 冲积扇：`/public/models/river/alluvial-fan.glb`
- 三角洲：`/public/models/river/delta.glb`
- 牛轭湖：`/public/models/river/oxbow-lake.glb`

请使用：

- Vite
- 原生 HTML、CSS、JavaScript
- Three.js
- `GLTFLoader`
- `OrbitControls`

不要使用 React、Vue 或后端服务。项目应可通过 `npm install` 和 `npm run dev` 直接运行。

# 一、核心目标

做出一个类似“3D 生物细胞图鉴馆”的三栏互动教学网站，但主题改为“河流地貌”。

必须实现以下完整链路：

1. 页面默认加载 V 型谷 GLB 模型。
2. 用户点击左侧地貌列表时，中间区域平滑切换对应的 GLB 模型。
3. 模型支持鼠标拖拽旋转、滚轮缩放、右键或双指平移。
4. 切换模型后，中间标题、副标题、右侧知识详情自动更新。
5. 用户点击“自动旋转”后，模型开始或停止缓慢旋转。
6. 用户点击“重置视角”后，相机和模型恢复默认观察角度。
7. 模型加载期间显示精致的加载状态。
8. GLB 文件加载失败时显示明确、友好的错误提示，而不是白屏。
9. 页面需要响应式适配桌面端和较窄屏幕；窄屏时左右栏可折叠或改为上下布局。

第一版暂时不要实现真实物理模拟、复杂演化动画、用户登录、后端数据库、地图 API、热点点击或答题系统。代码中预留扩展位置即可。

# 二、整体视觉风格

整体风格参考高级、柔和、干净的教育类 3D 展示平台，不要使用深色科技风。

视觉关键词：

- 温暖奶油白背景
- 柔和卡片阴影
- 浅米色、鼠尾草绿、雾蓝色、浅砂岩色
- 圆角面板
- 留白充分
- 简洁高级的知识图鉴感
- 让 3D 模型成为视觉中心
- 不要廉价渐变、不要过度炫光、不要花哨背景

建议配色：

## 3. 中间 3D 模型观察区

中间是页面视觉核心。

顶部动态显示：

- 地貌中文名称，例如：`V 型谷`
- 英文名称，例如：`V-shaped Valley`
- 一句副标题，例如：`河流长期下切侵蚀形成的典型侵蚀地貌。`

Three.js Canvas 放在中间大卡片中，背景透明或使用极浅米白色。

模型展示要求：

- 使用透视相机。
- 使用 OrbitControls。
- 禁止模型被拖出太远。
- 设置合理的最小和最大缩放距离。
- 鼠标拖拽可以旋转。
- 滚轮缩放。
- 保持模型居中。
- 切换模型时自动计算包围盒，并让相机自动适配模型尺寸。
- 模型加载完成后有轻微淡入动画。
- 不同模型尺寸可能不同，必须自动归一化到相近显示尺度。

场景灯光要求：

- 环境光。
- 柔和方向光。
- 模型下方带柔和投影。
- 使用浅色背景，不要黑色天空盒。
- 让模型保留原始材质、贴图和颜色。

模型区域右上角添加四个圆角小图标按钮：

- 旋转模式提示
- 缩放提示
- 全屏模式
- 截图按钮

其中：

- 全屏模式：让中间 3D 面板进入浏览器全屏。
- 截图按钮：将当前 Three.js canvas 导出为 PNG 下载。
- 旋转和缩放按钮可以仅显示简短 Tooltip，不需要额外功能。

模型区底部居中添加操作提示：

`拖拽旋转 · 滚轮缩放 · 点击按钮重置视角`

底部工具栏放两个主要按钮：

- `自动旋转`
- `重置视角`

自动旋转按钮再次点击时切换为“停止旋转”。

# 四、交互细节

1. 左侧菜单切换时，当前项目高亮。
2. 模型切换时：
   - 先显示半透明 loading overlay；
   - 加载完成后淡入；
   - 更新标题、右栏数据、学习提示；
   - 自动执行相机适配。
3. 自动旋转默认关闭。
4. 重置视角应重置相机位置、OrbitControls target、模型旋转。
5. 页面加载时默认选中 V 型谷。
6. 点击“对比实验室”“学习笔记”时显示 Toast：
   `该模块将在下一版本开放。`
7. 页面不能出现控制台报错。
8. 所有主要按钮应有 hover、active 和 Tooltip 或 title。
9. 加载 GLB 时使用 `LoadingManager` 或可见进度文字，例如：
   `正在加载 V 型谷模型…`
10. 如果路径错误，显示：
   `模型暂时无法加载，请检查 public/models/river/ 中的文件名与路径。`

# 五、文件结构要求

请按下面的目录结构创建项目：

```text
geoform-evolution-lab/
├─ index.html
├─ package.json
├─ vite.config.js
├─ src/
│  ├─ main.js
│  ├─ scene.js
│  ├─ modelLoader.js
│  ├─ ui.js
│  ├─ data/
│  │  └─ landforms.js
│  └─ styles/
│     └─ main.css
└─ public/
   └─ models/
      └─ river/
         ├─ v-valley.glb
         ├─ alluvial-fan.glb
         ├─ delta.glb
         └─ oxbow-lake.glb
```

代码职责建议：

- `main.js`：初始化页面、状态管理、菜单点击事件。
- `scene.js`：Three.js 场景、相机、光照、渲染器、OrbitControls、resize。
- `modelLoader.js`：GLB 模型加载、旧模型销毁、归一化缩放、居中、相机适配。
- `ui.js`：右侧信息渲染、左侧菜单状态、Toast、加载状态、按钮状态。
- `landforms.js`：保存四种地貌数据。
- `main.css`：全部页面布局与视觉样式。

# 六、代码质量要求

请确保：

- 使用模块化 JavaScript。
- 不把所有代码塞进一个文件。
- 模型切换时正确 dispose 旧模型的 geometry、material、texture，避免内存泄漏。
- 对所有 DOM 查询进行空值保护。
- 窗口缩放时 Three.js canvas 正确更新。
- 所有按钮可访问，至少有 `aria-label` 或 `title`。
- UI 文案全部使用中文。
- 代码中加必要中文注释，方便后续继续开发。
- 不要生成虚假的模型文件。
- 若 GLB 文件不存在，保留错误提示逻辑即可。
- 在 README 中说明如何放置模型、如何运行项目、如何后续新增一个地貌。

最后请输出完整可运行代码，并按文件分别展示代码内容。
